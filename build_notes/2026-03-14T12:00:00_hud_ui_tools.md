# Build Notes: HUD & UI Tools

**Date:** 2026-03-14
**Task:** WEB-02 (Tool interaction — click to place) + WEB-04 (HUD and status panel)
**Status:** Complete (mock mode)

## What was built

Three new modules in `crates/groundwork-web/src/ui/`:

### 1. Raycaster (`ui/raycaster.ts`)
- Converts mouse screen coordinates to voxel grid coordinates via Three.js Raycaster
- Handles face normal detection to determine adjacent vs. hit voxel
- `placingMode` parameter: place tools target adjacent voxel, shovel targets hit voxel
- Clamps to grid bounds (0..GRID_X-1, 0..GRID_Y-1, 0..GRID_Z-1)

### 2. HUD (`ui/hud.ts`)
- HTML/CSS overlay injected via JS (no index.html changes needed)
- **Tool palette** (bottom center): Shovel, Seed, Water, Soil, Stone — 5 buttons with active state
- **Species picker** (slides up above tool bar when Seed tool is active): 12 species grouped by type (Tree, Shrub, Flower, Ground)
- **Status display** (top-left): tick counter + auto-tick state
- **Help text** (top-right): control hints
- **Tick toggle** button
- Visual style: warm, dark glass-morphism panels matching the game's cozy aesthetic
- `Hud` class exposes state change callback for integration

### 3. Controls (`ui/controls.ts`)
- Keyboard shortcuts: `1-5` for tool selection, `Q/E` for species cycling
- Click-to-place: distinguishes clicks from drags using distance threshold (5px) and time limit (300ms)
- Prevents click-through: clicks on HUD elements don't place tools
- `onToolPlaced` callback for triggering re-mesh

### 4. Main integration (`main.ts`)
- Mock mode tool placement: modifies the local `Uint8Array` grid directly
- Automatic chunk re-meshing after tool placement via `ChunkManager.detectChanges()`
- Spacebar auto-tick syncs with HUD display
- All existing camera orbit/zoom input preserved

## Architecture decisions

- **HTML/CSS overlay** (not Three.js sprites): crisp text, standard hover/click behavior, accessibility, easy to style
- **Click vs. drag detection** in controls module: prevents accidental tool placement during camera orbit
- **Mock mode fallback**: tools work locally on the `Uint8Array` grid even without WASM. When WASM bridge is connected, `placeTool()` goes through bridge instead.
- **Species picker only shows when seed tool is active**: reduces visual clutter
