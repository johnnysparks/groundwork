# Player -> Manager Handoff: Tick Panic Blocks Playtest

## Observed
- Web UI launch attempt failed in this environment with `listen EPERM` on `127.0.0.1:5173`.
- CLI fallback works for `new`, `status`, `inspect`, and `place`.
- `tick` panics 100% of the time on a fresh world (`new` then `tick 1`) with Bevy ECS validation error: missing resource.
- Panic also reproduces after seed placement.
- Because tick never advances, no growth, fauna, dispersal, or interaction chain can be observed.

## Felt
- Session confidence collapses immediately after first tick crash.
- Current experience feels blocked, not rough.

## Bugs
- **BUG (Blocker):** `tick` panics due missing ECS resource (`Resource does not exist`), 100% repro.
- **BUG (Major, env-constrained):** web dev server cannot bind port in this sandbox (`EPERM`), forcing CLI fallback.

## Confusions
- Missing-resource panic message does not identify responsible system/resource in this build config.
- `place water` failure message is generic (`Nothing placed`) with no actionable reason breakdown.

## What made me want to keep playing
- Inspect output shows meaningful soil data and suggests good underlying simulation depth.

## What made me want to stop
- Immediate tick panic prevents any playable progression.

## Requests
1. Prioritize a P0 fix for missing-resource tick panic so sessions can run past tick 0.
2. Add non-debug diagnostics for system/resource names when ECS validation fails.
3. If possible for automation environments, provide a non-networked web playtest mode or documented headless rendering path that avoids local port binding.
